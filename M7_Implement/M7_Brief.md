### Required Documentation

For the final project, you will need to submit the following:

1. **Project Report** using one of the templates provided (10-12 pages single spaced or 15-20 double spaced, for ~3,500 words at minimum)
2. **Solution Design Document** for the associated source code
3. **Code Repository / Online Hosted Notebook**

You can use any of the strategies you have learned about in the previous 9 courses as the basis for your project. Make sure that the selected method provides you with the results you will need in order to answer your initial problem statement. Further, make sure that your Project Report uses an academic template and follows academic reporting guidelines, clearly discusses all aspects of the project, and represents the results in a clear and concise way.

#### Solution Design Document Requirements

In addition to your Project Report, you must include a **one-page document** that lists the project specifications and solution architecture (Solution Design Document). This document should:

- Be understandable by someone with **no programming background**
- Imagine it as a business case for your solution that needs to be signed off by your manager before you can build and deploy it
- List out specific areas of the code and what each is meant to address/achieve

> Remember: Though your code would be very familiar to you, it is very difficult for a third person to directly read the code and understand it.

#### GitHub Repository Requirements

If you are using Python scripts and have shared an online repository of code files, then you are also required to:

- Submit all of your source code and a README file via a shared GitHub project so that your instructor is able to clone the repository and run your project locally
- Include a `requirements.txt` file for Python which would help the instructor install all required packages using: `pip install -r requirements.txt`
- This document must contain any specific instructions for how your program should be executed

> **Note**: If you have used a hosted Python notebook, then this step can be ignored.

Finally, your source code should be clear, fully commented, and optimized as best as possible. Refer to the Capstone Source Code Development Guidelines published in the Course Overview of your online course room for further details.
